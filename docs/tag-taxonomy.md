## Tag Usage Guidelines

### 1. Primary Tags (2-4 per lesson)
Every lesson should have:
- **Skill Category Tag**: One tag from primary skill categories above
- **Activity Tag**: `Lab` if hands-on, otherwise topic-specific
- **Difficulty Indicator**: Implicit from difficulty level, explicit if needed
- **Topic Tag**: Specific topic area

### 2. PL-300 Alignment
Lessons covering PL-300 exam topics should include:
- `PL-300` tag
- Relevant domain tags (e.g., `Prepare Data`, `Model Data`, `Visualize Data`, `Manage/Secure`)

### 3. Feature Tags
- Use `2025 Features` for new capabilities from 2025 updates
- Use specific feature tags (e.g., `Copilot`, `Visual Calculations`) when applicable

### 4. Critical Concepts
Use `Critical Concept`, `Critical Function`, or `Critical Decision` for:
- Fundamental concepts that students must master
- Essential functions (like CALCULATE)
- Architectural decisions with lasting impact

### 5. Consistency Rules
- **Capitalization**: Use Title Case for multi-word tags
- **Plurals**: Use singular form (e.g., `Chart` not `Charts`)
- **Abbreviations**: Use sparingly, only when widely recognized (e.g., `RLS`, `OLAP`)
- **Hyphens**: Use for compound concepts (e.g., `Row-Level Security`, `Year-over-Year`)

### 6. Tag Combinations
Common tag combinations:
- DAX lessons: `DAX` + `Measures` or `Calculated Columns` + difficulty
- Lab lessons: `Lab` + skill category + `Hands-On Lab`
- Service lessons: `Power BI Service` + specific feature
- Performance: `Performance` + specific technique
